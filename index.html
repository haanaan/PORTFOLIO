AWOC

Dépôt Github

Présentation

L'application AWOC, ou A Week Of Culture, est une application web conçue pour publier un contenu culturel chaque semaine. Accessible à tous les membres de l'établissement Gabriel Fauré, cette application permet diverses interactions. Les utilisateurs peuvent non seulement poster leurs propres contenus, mais aussi interagir avec les publications des autres en laissant des commentaires ou en mettant des "likes".

Méthodes de travail

Le projet a été développé par les sept étudiants de la classe de BTS SIO 2 option SLAM en suivant une méthodologie agile. Le but principal de ce projet a été d'appendre à répondre aux besoins potentiels des utilisateurs et de mettre régulièrement à jour l'application pour une amélioration continue. Pour la création de l'application web, ont été utilisés le framework PHP Symfony, accompagné de JavaScript pour les fonctionnalités dynamiques, et HTML, CSS et Bootstrap pour la structure et le design. La gestion des données est assurée par une base de données relationnelle SQL, administrée via PHPMyAdmin. Un système de user story a également été mis en place pour organiser les tâches de chacun.

Compétences développées

Compétence Description
Travailler en mode projet Utilisation exclusive de la méthode AGILE SCRUM, incluant des daily scrums, un backlog avec des user stories, et une livraison itérative du produit.
Répondre aux besoins Ajout continu de nouvelles demandes au backlog. Chaque développeur choisit une demande, la comprend et trouve une solution viable puis la met en place.
Faire des recherches Confrontation à de nombreux problèmes inconnus nécessitant des recherches personnelles et un travail en autonomie.
Utilisation de branches Utilisation de GitHub pour gérer le code. Chaque développeur crée des branches pour les user stories, puis fusionne les branches avec la principale en résolvant les conflits éventuels.
GSB Frais

Documentation

Présentation

L'application web GSB Frais a été développée pour gérer les frais de déplacement des visiteurs médicaux d'une entreprise fictive, GSB. Cette application permet d'enregistrer toutes les dépenses, y compris les frais de déplacement, de restauration et d'hébergement, ainsi que les coûts associés aux activités annexes telles que les événements et les conférences. Elle offre également un suivi détaillé et daté des opérations effectuées par le service comptable, depuis la réception des justificatifs jusqu'au remboursement des frais.

Méthodes de travail

Dans le cadre du projet, une nouvelle application web Symfony a été développée pour améliorer la gestion des frais des visiteurs médicaux. Cette initiative faisait suite à une application précédente nécessitant des améliorations. Pour ce faire, nous avons optimisé le code de l'ancienne application.
Nous avons utilisé l'environnement de développement intégré (IDE) PhpStorm et suivi l'architecture Modèle-Vue-Contrôleur (MVC). Le développement a impliqué les langages PHP, JavaScript, HTML, CSS, ainsi que Bootstrap pour le design. Pour la gestion de la base de données, nous avons utilisé PHPMyAdmin pour interagir avec le système de gestion de base de données SQL.

GSB Visites

Documentation

Présentation

L'application Android GSB Visites a été développée pour gérer les visites des visiteurs médicaux chez les praticiens de l'entreprise fictive GSB. Cette application permet aux utilisateurs d'accéder à des informations sur les praticiens, les visites et les visiteurs du laboratoire. Elle offre comme fonctionnalités l'authentification des visiteurs, l'enregistrement des comptes-rendus de visite, incluant la date, le motif de la visite, un commentaire, ainsi que les détails du visiteur et du praticien associés, et enfin l'affichage des détails des praticiens liés à chaque visiteur connecté.

Méthodes de travail

L'application est développée pour l'environnement OS Android en utilisant le langage de programmation Java et l’IDE Android Studio.Elle respecte les conventions de l'architecture POO. Pour assurer la sécurité des données, une authentification des utilisateurs est nécessaire pour accéder à ses fonctionnalités.
Côté serveur, une API a été créée à l'aide du framework Node.js, notamment Express.js. Cette API définit des routes répondant à des requêtes HTTP spécifiques (GET, POST, PUT, DELETE) et renvoie des données formatées en JSON à l'application mobile. Les données sont stockées dans une base de données NoSQL MongoDB.

Compétences développées

Compétence Description
Analyser les exigences Capacité à comprendre les besoins des utilisateurs et à traduire ces exigences en fonctionnalités concrètes pour l'application Android.
Conception de l'interface utilisateur Capacité à concevoir des interfaces utilisateur intuitives et esthétiques en suivant les maquettes crées pour l'application.
Gestion des API REST Flexibilité pour s'adapter aux changements et aux nouvelles exigences du projet tout en maintenant la cohérence et la qualité du travail réalisé.
Test et débogage Utilisation de périphériques Android réels pour tester et déboguer l'application afin d'assurer sa compatibilité et sa stabilité sur différents appareils. Utilisation du programme postman pour tester les requêtes de l'api.
Travailler en autonomie Capacité à travailler de manière autonome sur un projet de longue durée, en définissant et en respectant les objectifs, les échéances et les priorités. Tout cela en faisant preuve de persévérance pour surmonter les obstacles rencontrés.

Stage 2ème année

Compte rendu

Présentation

L'association Mots et Merveilles est une association qui forme des bénévoles et propose des cours particuliers, des ateliers culturels, citoyens et numériques aux personnes en situation d'illettrisme. Elle compte 12 sites dans les Hauts de France. Au cours de mon stage de 6 semaines j'ai eu l'occasion de participer à la maintenace et l'amélioration d'une application web utilisée par les employés et bénévoles de l'association. Mon rôle était donc de gérer l'application et de répondre aux besoins des utilisateurs sur le site d'Aulnoyes-Aymeries.

Méthodes de travail

Pendant mon stage, mon rôle principal consistait à analyser et à rendre compte des besoins des utilisateurs, ainsi que des problèmes qu'ils rencontraient. Une fois ces problèmes identifiés, je les présentais à la directrice adjointe et à mon responsable informatique lors de réunions dédiées, où nous discutions des solutions adaptées. Ensemble, nous élaborions des user stories pour décrire de manière claire et non technique les améliorations nécessaires. Ensuite, j'étais chargé de mettre en œuvre ces améliorations dans l'application web Symfony.
Une fois les changements réalisés, ils étaient soumis à l'approbation de mon supérieur informatique et de la directrice adjointe. Une fois validés, les améliorations étaient déployées sur l'application principale, bien que je travaillais sur une version de test de l'application. Durant tout le processus, je communiquais régulièrement avec mes collègues et je prenais des notes sur tous les problèmes rencontrés pour assurer un suivi efficace.
